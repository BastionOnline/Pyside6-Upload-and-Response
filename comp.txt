In your MyApp.spec

Look for the EXE(...) call at the bottom. You’ll see something like:

exe = EXE(
    ...,
    name='MyApp',
    debug=False,
    bootloader_ignore_signals=False,
    strip=False,
    upx=True,
    console=True,
)


And above that, there’s a COLLECT(...) line if you originally built with onedir.

To force onedir build

Make sure the bottom part of the spec ends with COLLECT, like this:

coll = COLLECT(
    exe,
    a.binaries,
    a.zipfiles,
    a.datas,
    strip=False,
    upx=True,
    upx_exclude=[],
    name='MyApp'
)


That tells PyInstaller:

Put the exe in a folder with all dependencies (MyApp/dist/MyApp/).

Much faster to build and run, and easier for debugging.




To force onefile build

Instead of COLLECT, it wraps with EXE directly. Example:

exe = EXE(
    pyz,
    a.scripts,
    a.binaries,
    a.zipfiles,
    a.datas,
    [],
    name='MyApp',
    debug=False,
    bootloader_ignore_signals=False,
    strip=False,
    upx=True,
    console=True,
